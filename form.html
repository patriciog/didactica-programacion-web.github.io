5. Formularios: <form> e interactividad

Los formularios son la herramienta principal para que los usuarios interactúen con tu sitio web enviando datos. Se definen mediante la etiqueta <form>, que actúa como un contenedor para todos los campos de entrada. El atributo "action" especifica a dónde se envían los datos, mientras que "method" indica cómo se envían al servidor.

Para que un formulario sea profesional y accesible, siempre debemos usar la etiqueta <label> vinculada a cada campo. El elemento <input> es el más versátil, ya que su función cambia totalmente según el atributo "type" que le asignes. Existen tipos específicos para texto simple, correos electrónicos, contraseñas, fechas y mucho más.

HTML5 introdujo validaciones automáticas, permitiendo que el navegador verifique si un campo obligatorio está vacío antes de enviarlo. El atributo "name" es fundamental, pues es el nombre que el servidor buscará para procesar la información recibida. Además de los inputs, podemos usar <textarea> para textos largos y <select> para menús desplegables.

Finalmente, todo formulario necesita un botón de envío, que suele ser un <button> o un input de tipo "submit". Dominar los formularios es esencial para crear aplicaciones web funcionales. En nuestra última lección, aprenderemos a insertar contenido externo en nuestra página.
